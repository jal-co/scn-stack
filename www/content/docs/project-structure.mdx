## Overview

After running `npx create-scn-stack`, you'll get a fully configured project. Here's the structure for a **Next.js + Fumadocs** setup with essentials:

```
my-ui/
├── app/
│   ├── layout.tsx               # Root layout with Fumadocs provider
│   ├── page.tsx                 # Landing page
│   ├── globals.css              # Tailwind v4 + shadcn CSS variables
│   └── docs/
│       ├── layout.tsx           # Docs sidebar layout
│       └── [[...slug]]/
│           └── page.tsx         # Dynamic docs page
├── content/docs/                # Documentation content (MDX)
│   ├── meta.json                # Navigation structure
│   ├── index.mdx                # Getting started
│   ├── installation.mdx         # Install instructions
│   └── components/
│       ├── meta.json            # Component nav order
│       ├── button.mdx           # Button documentation
│       ├── card.mdx             # Card documentation
│       └── badge.mdx            # Badge documentation
├── registry/new-york/ui/        # Component source files
│   ├── button.tsx               # Button with CVA variants
│   ├── card.tsx                 # Card compound component
│   └── badge.tsx                # Badge with CVA variants
├── lib/
│   ├── source.ts                # Fumadocs content loader
│   └── utils.ts                 # cn() utility
├── public/r/                    # Built registry output (generated)
│   ├── registry.json
│   ├── button.json
│   ├── card.json
│   └── badge.json
├── registry.json                # Registry definition (source of truth)
├── components.json              # shadcn configuration
├── source.config.ts             # Fumadocs MDX config
├── next.config.ts               # Next.js + MDX plugin
├── tsconfig.json                # TypeScript config
├── postcss.config.mjs           # Tailwind v4 PostCSS
└── package.json                 # Scripts and dependencies
```

## Key Files

### registry.json

The source of truth for your registry. Defines every item (component, hook, block) that can be installed via the shadcn CLI.

```json
{
  "$schema": "https://ui.shadcn.com/schema/registry.json",
  "name": "my-ui",
  "homepage": "https://my-ui.com",
  "items": [
    {
      "name": "button",
      "type": "registry:ui",
      "title": "Button",
      "description": "A button component with multiple variants.",
      "dependencies": ["class-variance-authority"],
      "files": [
        {
          "path": "registry/new-york/ui/button.tsx",
          "type": "registry:ui"
        }
      ]
    }
  ]
}
```

### components.json

The shadcn configuration for your project. Includes your style, aliases, and namespace.

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "rsc": true,
  "tsx": true,
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  },
  "iconLibrary": "lucide",
  "registries": {
    "@my-ui": "https://my-ui.com/r/{name}.json"
  }
}
```

### public/r/ (Generated)

Static JSON files generated by `shadcn build`. This is what the shadcn CLI fetches when users install your components. **Don't edit these directly** — they're generated from `registry.json` and your source files.

## Adding Components

1. Create your component in `registry/<style>/ui/<name>.tsx`
2. Add the item to `registry.json`
3. Run `pnpm registry:build`
4. Add a docs page in `content/docs/components/<name>.mdx`

## Scripts

| Script | Description |
|--------|-------------|
| `dev` | Start development server |
| `build` | Build for production (includes registry build) |
| `registry:build` | Generate static registry JSON files |
| `start` | Start production server |
| `lint` | Run ESLint |
