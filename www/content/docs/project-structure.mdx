## Key Files

### registry.json (Root)

The root registry uses the `include` pattern to compose from per-directory files:

```json
{
  "$schema": "https://ui.shadcn.com/schema/registry.json",
  "name": "my-ui",
  "homepage": "https://my-ui.com",
  "include": [
    "registry/new-york/ui/registry.json"
  ]
}
```

### registry/\<style\>/ui/registry.json

Per-directory registry defining each component. File paths are relative to this file:

```json
{
  "$schema": "https://ui.shadcn.com/schema/registry.json",
  "items": [
    {
      "name": "button",
      "type": "registry:ui",
      "title": "Button",
      "description": "A button component with multiple variants.",
      "dependencies": ["class-variance-authority"],
      "files": [
        { "path": "button.tsx", "type": "registry:ui" }
      ]
    }
  ]
}
```

### components.json

The shadcn configuration for your project. Includes your style, aliases, and namespace.

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "rsc": true,
  "tsx": true,
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  },
  "iconLibrary": "lucide",
  "registries": {
    "@my-ui": "https://my-ui.com/r/{name}.json"
  }
}
```

### Component Previews

Each component doc page includes a live preview using `ComponentPreview` and demo files that import directly from the registry source:

```tsx
import { ComponentPreview } from "@/components/component-preview"
import { ButtonDemo } from "@/components/examples/button-demo"

<ComponentPreview>
  <ButtonDemo />
</ComponentPreview>
```

### AI Skills

If enabled, the project includes an `.agents/skills/registry/SKILL.md` that teaches AI assistants (Claude, Cursor, Copilot) how to work with your specific registry — adding components, writing docs, building, and following conventions.

### public/r/ (Generated)

Static JSON files generated by `shadcn build`. This is what the shadcn CLI fetches when users install your components. **Don't edit these directly** — they're generated from `registry.json` and your source files.

## Adding Components

Use the CLI to scaffold a new component with one command:

```bash
npx create-scn-stack add-component input -d "A text input component."
```

This creates:
1. Component source in `registry/<style>/ui/<name>.tsx`
2. Entry in `registry/<style>/ui/registry.json`
3. Doc page in `content/docs/components/<name>.mdx`
4. Updates `meta.json` navigation

Or manually:

1. Create your component in `registry/<style>/ui/<name>.tsx`
2. Add the item to `registry/<style>/ui/registry.json`
3. Run `pnpm registry:build`
4. Add a documentation page in `content/docs/components/<name>.mdx`

## Validate & Build

```bash
npx shadcn registry validate
pnpm registry:build
```

## Scripts

| Script | Description |
|--------|-------------|
| `dev` | Start development server |
| `build` | Build for production (includes registry build) |
| `registry:build` | Generate static registry JSON files |
| `start` | Start production server |
| `lint` | Run ESLint |
